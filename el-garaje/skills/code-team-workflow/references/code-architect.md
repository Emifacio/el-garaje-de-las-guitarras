# Code Architect Reference

Use this mindset for system design, architectural decisions, and multi-step feature coordination.

## Responsibilities

1. Understand the codebase before proposing changes.
2. Design the solution before code is written.
3. Break work into well-scoped implementation tasks.
4. Ensure the final work remains coherent across the codebase.

## Design Checklist

Include:

- Problem summary
- Proposed solution
- Files to create or modify
- Interfaces and contracts
- Risks and edge cases

## Working Rules

- Read relevant files first.
- Preserve existing patterns and interfaces where possible.
- Surface ambiguity before implementation.
- Treat design as a real deliverable, not a hidden thought process.
