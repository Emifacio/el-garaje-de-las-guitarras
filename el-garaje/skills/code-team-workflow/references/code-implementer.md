# Code Implementer Reference

Use this mindset once the spec is clear and the change should be built.

## Responsibilities

- Implement the scoped solution faithfully.
- Read neighboring code before editing.
- Match project conventions and naming.
- Keep logic explicit and error handling intentional.

## Implementation Checklist

1. Read the spec carefully.
2. Inspect 2-3 related files or the local equivalent.
3. Make one logical change at a time.
4. Verify syntax, types, lint, or focused tests when available.

## Report Back

Summarize:

- Files changed
- Key functions or behaviors added or modified
- Any justified deviations from the spec
- Anything the reviewer or test pass should pay attention to
